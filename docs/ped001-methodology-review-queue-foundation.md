# PED-001 — Methodology Config + Review Queue Foundation

## Goal

Create the pedagogical foundation for Boom.

This story does not yet update mastery states automatically. It creates the schema and domain model needed for methodology selection, reapproach frequency, skill mastery tracking, review queue, abandonment and reentry rules, correlated skill remediation, and AI-assisted interventions.

## Product decision

Boom is the source of truth.

The AI provider must not be the system of record for the student profile. Boom stores the state, methodology, mastery score, review queue and intervention history. AI receives a compact context and returns explanations, questions or recommendations.

## Tables

### `learning_methodologies`

Stores available methodologies and their configuration.

Seeded methodologies:

```text
LIGHT_REVIEW
BALANCED
INTENSIVE_REMEDIATION
EXAM_PREP
```

### `student_learning_strategy`

Stores the selected methodology and mode for each student.

### `student_skill_mastery_state`

Stores the current pedagogical state per student and skill.

### `student_review_queue`

Stores due, overdue and future reapproach tasks.

### `learning_skill_relationships`

Stores prerequisite, related and remediation-support relationships between skills.

### `student_learning_interventions`

Stores interventions generated by Boom or AI.
